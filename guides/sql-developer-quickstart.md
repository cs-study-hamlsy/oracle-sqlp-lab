# SQL Developer 빠른 시작

## 목적

이 저장소의 실습을 가장 쉽게 수행하는 방법은 Oracle SQL Developer에서 교재별 주제 폴더 안의 SQL 파일을 순서대로 실행하는 것입니다.

## 권장 방식

1. Oracle SQL Developer에서 실습 계정으로 접속합니다.
2. 원하는 교재 폴더 안의 `파트 > 주제` 폴더로 이동합니다.
3. 주제 폴더 안의 `sql/` 디렉터리에서 각 `.sql` 파일을 엽니다.
4. `01_setup.sql`부터 번호 순서대로 `파일 전체`를 실행합니다.
5. 각 SQL의 `prompt`와 주석을 읽으며 실행계획을 비교합니다.

## SQL Developer에서 어떻게 실행할까

- `sql/01_setup.sql`, `sql/02_*.sql`, `sql/03_*.sql`, `sql/05_answer.sql`, `sql/99_cleanup.sql`은 원칙적으로 `파일 전체 실행`을 권장합니다.
- SQL Developer에서는 가능하면 `F5(스크립트 실행)`로 돌립니다.
- 이유는 이 저장소의 스크립트가 `prompt`, `PL/SQL 블록`, `/` 구분자, `explain plan`, `dbms_xplan.display` 흐름을 포함하기 때문입니다.
- `Ctrl+Enter`는 현재 커서의 한 문장만 실행하므로, 여러 문장을 순서대로 봐야 하는 실습에는 적합하지 않습니다.
- `sql/04_practice.sql`은 안내문 성격이 강하므로, 파일을 그대로 실행해도 되지만 실제 연습은 파일의 예시 SQL을 복사해서 별도 워크시트에서 수정하며 해보는 방식을 권장합니다.
- `sql/05_answer.sql`은 직접 풀어본 뒤 비교하는 정답 스크립트이므로, 가능하면 `04_practice.sql` 뒤에 실행하는 것이 좋습니다.

## 파일별 실행 원칙

- `sql/01_setup.sql`: 처음 한 번 전체 실행합니다.
- `sql/02_*.sql`: 파일 전체 실행해서 기준 실행계획을 확인합니다.
- `sql/03_*.sql`: 파일 전체 실행해서 힌트/조건 변경 결과를 비교합니다.
- `sql/04_practice.sql`: 문제를 읽고, 예시 SQL을 복사해 조건이나 힌트를 바꿔 직접 실행합니다.
- `sql/05_answer.sql`: 직접 풀어본 뒤 정답 SQL과 실행계획 비교 포인트를 확인합니다.
- `sql/99_cleanup.sql`: 실습 오브젝트를 정리하고 싶을 때만 전체 실행합니다.

## 추천 접속 계정

- 가능하면 `SYSTEM` 대신 별도 실습 계정을 사용합니다.
- 이유는 오브젝트 관리, 권한 통제, 실습 반복성 측면에서 더 안전하기 때문입니다.

## 실습 전 확인

- `DBMS_XPLAN.DISPLAY` 조회 가능 여부
- `PLAN_TABLE` 사용 가능 여부
- 샘플 스키마 또는 실습용 계정 준비 여부

## 자주 헷갈리는 점

- 이 저장소는 "파일 일부를 복사해서 붙여넣는 방식"보다 "교재의 주제 폴더 안 SQL 파일을 통째로 열어 순서대로 실행하는 방식"을 기본으로 합니다.
- 기본 위치는 `교재 폴더/파트 폴더/주제 폴더/sql/` 입니다.
- 다만 `sql/04_practice.sql`은 문제 안내용이므로, 그 안의 예시 SQL만 복사해서 직접 바꿔가며 실험하는 것이 더 좋습니다.
- 이후 `sql/05_answer.sql`을 실행하면서 내 SQL과 정답 SQL의 차이를 실행계획 기준으로 비교하면 복습 효율이 좋습니다.
- 실습 중 같은 `sql/01_setup.sql`을 다시 실행하면 기존 오브젝트를 지우고 다시 만들 수 있으므로, 이전 결과를 유지하고 싶다면 주의해야 합니다.

## 실습 중 꼭 볼 것

- `TABLE ACCESS FULL`
- `INDEX RANGE SCAN`
- `TABLE ACCESS BY INDEX ROWID`
- `NESTED LOOPS`
- `HASH JOIN`
- Predicate Information

## SQLP 관점 팁

- 힌트가 먹었는지 여부만 보지 말고, 왜 그 힌트가 유리하거나 불리한지도 설명해봐야 합니다.
- `Cost` 숫자만 외우지 말고 액세스 경로 변화와 조건 선택도를 함께 보아야 합니다.
